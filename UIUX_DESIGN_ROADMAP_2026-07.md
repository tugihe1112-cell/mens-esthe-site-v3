# メンエスマップ UI/UXデザイン改善ロードマップ（2026-07-05）

> コンサル文書。実装は別セッションで行う前提の「道筋」。
> SEO/機能ロードマップ（SITE_GRADEUP_ROADMAP_2026-07.md）は全消化済み → 次の伸びしろは「来た人を逃さない・信じさせる・書かせる」デザイン。
> 診断ソース: Home.jsx / ModernReviewCard.jsx / ThreadDetailPage.jsx / BottomNav.jsx / Header.jsx の実コード＋CLAUDE.mdのUI変遷履歴。

## 全体診断（結論から）

**構造は良い。問題は「情報の優先順位」と「信頼感の演出不足」の2点。**

1. **口コミファーストになっていない。** サイトの唯一無二の資産は「本物の口コミ」なのに、ホームでは6番目のセクション、セラピストページでは70vhの写真の下。ユーザー（＝「〇〇 口コミ」で検索して来る人）が見たいものが常にファーストビューの外にある。
2. **装飾過多で信頼感が薄い。** 全要素が `font-black`＋グラデーション＋絵文字アイコンで「叫んでいる」状態。口コミサイトの通貨は信頼。今のトーンは夜のポータルとしては正しいが、「レビューの中身は本物」という静かな説得力のレイヤーが足りない。
3. **細部の品質が7〜8割で止まっている。** 8〜10pxの日本語テキスト、英語ラベル残存（PROFILE DATA / No ratings yet / Loading...）、OS依存の絵文字アイコン、全画面テキストのローディング。1つ1つは小さいが「個人サイト感」の正体はここ。

**判断軸**: 「その変更は、①口コミを読む体験、②口コミを書く動機、③サイトへの信頼、のどれかを直接上げるか？」

## Tier A — 口コミの見せ方（資産価値を最大化・最優先）

### A-1. セラピストページの再設計（SEO着地ページ＝最重要テンプレ）
- **現状の問題**: `h-[70vh] md:h-[80vh]` の全画面写真 → プロフィールカード → やっと口コミ。検索流入者の目的（口コミ）が2スクロール先。
- **やること**（ThreadDetailPage.jsx）:
  - ヒーローを**コンパクトヘッダー化**: 写真は左40%（aspect-3/4・max-h-[320px]程度）、右に名前・店舗・評価サマリ（★平均・件数・タグ）。口コミ1件目がファーストビューに入る高さに。
  - **評価サマリバー**をヘッダー直下に: 「★3.2（3件の実体験レポ）」＋6軸の平均を横並びミニバーで。検索結果のリッチリザルトで見た★とページの見た目を一致させる（期待の整合＝直帰率低下）。
  - **パンくずUI**を表示（ホーム > 神奈川 > 店名 > 名前）。JSON-LDのBreadcrumbListは実装済みなので視覚版を揃えるだけ。
  - 英語ラベル一掃: PROFILE DATA→プロフィール / HEIGHT・SIZE・TYPE→身長・カップ・タイプ / Reviews→件 / No ratings yet→まだ評価がありません / Loading...・Therapist not found→日本語＋スケルトン。
  - ローディングを全画面テキスト→**スケルトン**（Skeleton.jsxに部品あり・流用）。

### A-2. 口コミカードの「本物らしさ」強化（ModernReviewCard.jsx）
- **現状の良い点**: 運営取材レポートバッジ・メータードペイウォール・コース表示は既に定石通り。
- **やること**:
  - **来店情報行を追加**: 「2026年6月来店・80分コース・総額2万円」を本文上部に小さく構造表示（courseフィールド流用＋来店時期フィールド）。実体験の証拠が最初に目に入る＝信頼の即時演出。
  - 6軸メトリクスのラベル9px→11px・数値をバー表示に（現在は数字のみで差が読み取れない）。
  - ★6個6pxの星列は視認不能 → 削除して数値バッジだけに（色分けemerald/amber/roseは維持・機能している）。
  - ロックCTAに**残り文字数**を出す: 「続き620文字は限定公開」→ 具体数字は好奇心を確実に上げる。`review.content.length - 140` を表示するだけ。
  - タグの色を系統別に分ける（体型=ピンク・雰囲気=パープル・年代=ブルー・属性=スレート）。現在全部同じピンクで情報として機能していない。

### A-3. ホームの「口コミファースト」化（Home.jsx）
- **現状の問題**: セクション順が 検索→W2R帯→機能カード→エリア→人気エリア→バナー→**最新の口コミ**→セラピスト→新着店舗。独自資産が6番目。
- **やること**:
  - **「最新の口コミ」を検索カード直下（2番目）へ移動**。カードも現状のテキスト2行→セラピスト写真サムネ付きに格上げ（口コミがビジュアルの主役になる）。
  - 「人気エリア」のUnsplash画像を廃止 → エリア名＋店舗数＋口コミ件数のタイポグラフィタイル に置換。**現状は無関係な外国の夜景で嘘くさく、しかも1.1MBのLCP負債**。画像なしタイルの方が速くて誠実。
  - 機能ショートカット4枚は3枚バナー（新人・口コミ・掲示板）と役割重複 → どちらかに統合して1画面分短縮。ホームは現在長すぎて下部セクションはほぼ見られていない（GA4滞在27秒）。

## Tier B — 書かせるUX（投稿転換率）

### B-1. セラピストページに追い CTA
- 口コミを読み終えた直後が投稿動機の最高点。**ページ下部スクロール時に出る sticky ボトムバー**「この人の口コミを書く → 7日間読み放題」を追加（PostReviewPageへ therapist/shop プリセット遷移は実装済み・入口を増やすだけ）。
- 口コミ0件ページの空状態を「先行者特典」演出に: 「最初のレポを書くと即7日間読み放題」（文言変更のみ）。

### B-2. 投稿フォームの心理設計（PostReviewPage.jsx）
- 文字数カウンターを**達成メーター化**: 0→200字（3日付与）→700字（7日付与）の2段階ゴールをプログレスバーで可視化し、しきい値通過時にマイクロ演出（色変化＋「3日分確定！」）。数字だけのカウンターより完走率が上がる定石。
- Step1の店舗コンボボックス: 入力中の**あいまい一致**（ひらがな/カタカナ正規化）が効くか確認。店名が探せない＝最大の離脱点。
- 各Stepに「あと◯ステップで投稿完了」の残数表示（現在はStep X/Xのみ）。

### B-3. 投稿後の体験
- 投稿完了画面を「おめでとう」だけでなく**即時価値の確認**に: 付与された閲覧日数・自分の口コミへのリンク・「読まれたら通知が届きます」（週次メール実装済み＝その予告）。投稿→定着の橋渡し。

## Tier C — 品質の底上げ（「個人サイト感」の除去）

### C-1. タイポグラフィの床を上げる
- **禁止ルールを作る**: 本文系は14px以上・補足は12px以上・10px未満は使用禁止（現在 text-[8px]〜[10px] が多数。BottomNavのラベル8px、カード内注釈9〜10px）。日本語は同サイズの英語より視認性が落ちる。
- `font-black` の使用を見出しのみに制限。本文・ラベルは `font-medium/bold` に落とし、太さで階層を作る（今は全部最太で階層が消えている）。

### C-2. アイコンの統一
- 絵文字アイコン（💃✨📋💬✍️🏆🔍🏢👤）はOS依存で見た目がバラつき、チープさの主因。**lucide系のインラインSVGに統一**（ModernReviewCard.jsxに自前Lucide互換SVGが既にある＝これを `src/components/icons.jsx` に切り出して全ページで使い回す。新規依存不要）。

### C-3. デザイントークン化（tailwind.config.js）
- 現在は色・角丸・ボーダーが各所ハードコード（rounded-2xl/3xl/[2.5rem]混在、border-white/5と/10混在）。
- `theme.extend` に集約: surface（slate-900）/ surface-2（slate-800）/ line（white/10）/ accent（pink-500）/ accent-2（purple-500）、radius は sm=12px・md=16px・lg=24px の3段だけに。**新しい見た目は増やさず、既存の揺れを3〜4パターンに収斂させる**のが目的。

### C-4. アクセシビリティ最低ライン
- `focus:outline-none` 単独使用の箇所に `focus-visible:ring` を追加(キーボード操作者を見捨てない)。
- slate-500 on slate-950 の本文はWCAG AA未達 → 補足テキストの下限を slate-400 に統一（Home.jsxは対応済み・他ページ残り）。
- `prefers-reduced-motion` でヒーロー自動回転と `animate-pulse` を停止するCSSを index.css に1ブロック追加。

## Tier D — できればやった方がいいこと（任意）

- **ダークテーマの「黒の質」改善**: 全面 slate-950 単色 → セクション境界に微グラデ（slate-950→#0a0614系の紫黒）を1枚敷くと安っぽさが消える。CSS1行の低コスト高印象。
- **OGP/faviconの再点検**: ブランド指名検索が増え始めるタイミングでSERPとシェア時の顔を整える（既ロードマップの積み残し）。
- **口コミカードのコピー防止（onCopy/user-select禁止）の緩和検討**: 転載防止と引用シェア（=被リンク源）はトレードオフ。ウォーターマークがあるので選択可否だけ緩めるのも一案。判断はデータが増えてから。
- **年齢確認モーダル**: 法的必須ではないが、業界慣習として入口に置くと広告審査・ストア展開時に有利。SEOへの影響（クローラー除外設定必須）とトレードオフなので今はやらない判断でOK。
- **ヒーローのcoverflowスライダー**: ユーザー好みで維持と決定済み（差し戻し履歴あり）。触らない。CLS 0.165も許容済み。

## 測定方法（デザイン改修の答え合わせ）

- GA4に `begin_review`（投稿Step1到達）と `complete_review`（投稿完了）イベントを追加 → **投稿ファネル転換率**が改修前後で比較できるように（trackEvent基盤は実装済み・1時間仕事）。
- セラピストページの**直帰率と滞在時間**をA-1前後で比較（GA4のページパス別レポート）。
- ロックCTAのクリック（`click_paywall_cta`）もイベント化 → A-2の残り文字数表示の効果測定。
