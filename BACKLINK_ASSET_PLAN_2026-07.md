# 被リンク資産づくり計画（2026-07-05）

> コンサル文書。実装は別セッション（Opus）＋配布はokabayashiの運用作業。
> 位置づけ: SEO/UIUXロードマップ消化後に残った**唯一の順位レバー＝ドメイン権威（被リンク）**への打ち手。
> 現状: 表示の94.7%が4位以下。コンテンツ・構造・UXは仕込み済み → 9位→3位の壁を超えるのに足りないのは外部評価。

---

## 戦略（結論から)

**「リンクしたくなる一次データ」を1本作り、それを配布する。**

個人ブログ・まとめサイト・Xの投稿者がメンエス記事を書くとき、引用できる統計データは現状ほぼ存在しない（各ポータルは店舗一覧のみ）。
うちには **shops約900店・therapists 45,000人・料金システム** という他に無い規模のDBがある。これを「メンエス統計白書」として公開すれば、
「出典: メンエスマップ調べ」付きで引用される＝自然リンクの獲得装置になる。

リンクは統計ページに集まるが、内部リンクでサイト全体（＝本命の口コミページ）にPageRankが流れる。

## Part 1 — 統計ページの設計（コード実装・Opus担当）

### 1-1. ページ仕様: `/stats`「日本のメンズエステ統計 2026」

**掲載する統計（DBから機械集計できるものだけ）:**

1. **都道府県別 店舗数ランキング** — shops を prefecture で集計。日本地図風の一覧＋TOP10バー
2. **エリア別 店舗密度TOP20** — raw_data.area 単位（例: 大宮・池袋・梅田…）。「メンエス激戦区ランキング」として引用されやすい見出しに
3. **料金相場** — `price_system` / `raw_data.system.standard[].price` から数値抽出し、都道府県別に60分/90分帯の**中央値**（平均は外れ値に弱い）。
4. **在籍セラピスト数の統計** — 総在籍数・1店舗あたり中央値・在籍数TOP10店舗
5. **（任意）新規出店トレンド** — shops.created_at はDB登録日で開業日ではないため「掲載店舗の推移」として表現を正確に

**信頼性ルール（must）:**
- 料金はデータが取れた店舗のみで集計し、**「調査対象N店舗」を必ず併記**。サンプル10店未満の都道府県は料金を出さない
- 「〜調べ（2026年7月時点・掲載N店舗のデータより）」の出典表記をページ内に固定表示
- 誇張・断定しない（景表法とサイト信頼の両方）

### 1-2. 実装方式

- **集計スクリプト**: `scripts/metrics/build_stats.mjs` — service roleでDB全件集計 → `src/data/stats-2026-07.json` を生成してコミット。月1回再実行して更新（ページに「最終更新日」表示）
  - 料金パース: `price.replace(/[^0-9]/g,'')` 系は既存ShopListPage.backupに前例あり。「60分」「90分」等のコース分数もテキストから抽出、パース不能はスキップしてカバレッジを出力
- **ページ**: `pages/stats.jsx` — JSONを静的import（DBアクセス不要＝高速・落ちない）。グラフは**CSSバー＋SVGで自前描画**（Chart.js等の依存追加はしない。既存のスタイルトーンに合わせる）
- **SEO**: title「【2026年版】メンズエステ統計｜店舗数・料金相場・激戦区ランキング」/ JSON-LDに`Dataset`/ sitemap追加 / ホームのフッターとサイドバーからリンク
- **引用されやすくする仕掛け**:
  - 各統計ブロックに「この数字を引用する」ボタン → 「出典: メンエスマップ（https://www.mens-esthe-map.jp/stats）」付きテキストをコピー
  - OGP画像はハイライト数字入り（既存 `api/og.js` を流用してstats用パラメータ追加）
  - 主要データはページ末尾に**表（table要素）**でも掲載（コピペ・スクレイプされやすい形＝引用されやすい形）

### 1-3. Opusへの実装指示文（コピペ用）

```
CLAUDE.mdとplaybook/を読んでから、リポジトリ直下の BACKLINK_ASSET_PLAN_2026-07.md の Part 1 を実装して。

手順:
1. scripts/metrics/build_stats.mjs を作成（service role・--dry-run対応）。
   まずdry-runで料金データのカバレッジ（パース成功率・都道府県別サンプル数）を出力し、
   サンプル10店未満の県は料金統計から除外するロジックを入れる
2. 集計結果を src/data/stats-2026-07.json に出力
3. pages/stats.jsx を新規作成（JSONを静的import・SSR不要・CSSバー/SVGで自前描画・依存追加禁止）
4. JSON-LD Dataset・sitemap追加・フッター/ホームサイドバーからのリンク・引用コピーボタン・
   「調査対象N店舗・2026年7月時点」の出典表記
5. npm run build 通過 → コミット → デプロイ → CLAUDE.mdログ更新

注意: 数字の誇張表現は禁止（「日本最大級」等はデータで言える範囲のみ）。
ヒーローcoverflow等の既存ページには触らない。
```

## Part 2 — 配布戦略（コード外・okabayashiの運用作業）

作っただけのページはリンクされない。**配布が本体**。優先順に：

### 2-1. X（Twitter）運用 — 統計の切り出し投稿
- アカウント設計: サイト公式として開設（プロフィールにサイトURL・「全国900店舗・45,000人のデータでメンエスを可視化」）
- 投稿テンプレ: 統計1個＝1ポスト。「【調べてみた】メンエス店舗数が多い駅TOP10、1位は◯◯でした（全国◯店舗集計）」＋ランキング画像＋statsページへのリンク
- 頻度: 週2〜3本で十分。統計切り出し→口コミ紹介→新着情報のローテ
- 狙い: まとめ系アカウント・ブログ運営者の目に入ること（フォロワー数ではなく「引用される1本」）

### 2-2. 店舗への連絡（BtoB種まきと兼用）
- 口コミを投入済みの店舗（ユニゾン・こころ・広島人妻さん）に「口コミページができました」+統計ページを添えて連絡
- 店舗が自サイト/Xでリンクしてくれれば1本目の被リンク。将来の広告営業の接点にもなる
- テンプレ: 「お客様の実体験レポートを掲載しています。掲載内容に問題があればお知らせください。あわせて全国統計も公開しました」→ 低姿勢・削除窓口の明示が信頼につながる

### 2-3. 相互リンク・寄稿の地道な獲得
- メンエス系個人ブログ（体験談ブログ）に統計データの利用を打診（「データ無償提供・出典リンクのみお願い」）
- 5ch/したらばのメンエススレは直接投稿NG（ステマ・宣伝扱い）。Xとブログ経由の自然発見に任せる

### 2-4. 測定
- GSCの「リンク」レポートで被リンク数を月次確認（metrics-logに追記）
- statsページへの流入とreferralをGA4で確認

## やらないこと（明確化）

- **プレスリリース配信（PR TIMES等）**: 有料3万円〜でアダルト隣接ジャンルは審査落ちリスク。ROI不明のため封印
- **リンク購入・相互リンク集**: ペナルティリスク。絶対にやらない
- **統計の水増し・推計値の混入**: 実DBで言える範囲だけ。信頼が資産
