# メンエスマップ グレードアップ戦略ロードマップ（2026-07-02）

> コンサル文書。実装は別セッションで行う前提の「道筋」だけをまとめたもの。
> 前提数値（07-01時点）: click 62 / 表示 1,656 / 平均順位 17.5 / GA4 日本人U 64・eng 60%。
> 本命3ページ（unison相模原 120表示@9位・こころ大阪 30@9.6・広島人妻 60@8.7）は6/22投入口コミの順位反映待ち。

## 全体診断（結論から）

サイトの構造・技術基盤はすでに個人開発として高水準（SSR・noindex制御・R2移行・計測自動化・投稿摩擦低減まで完了済み）。
**ボトルネックは一貫して「コンテンツ量と権威（被リンク）」であり、コード改善のROIは既に逓減している。**
よって優先順位は：

1. **Tier 0**: 完了済み施策の「取りこぼし回収」（未実行SQL・未テスト・未リクエスト）
2. **Tier 1**: コンテンツ増産の仕組み化（口コミ投入を週次ルーチン化）
3. **Tier 2**: SEO構造の底上げ（エリアページ・内部リンク・CTR改善）
4. **Tier 3**: フライホイール自走化（UGC獲得の導線・インセンティブ）
5. **Tier 4**: 技術基盤の残債（LCP・監視・テスト）
6. **Tier 5**: マネタイズ準備（Phase 3。前倒ししない）

## Tier 1 — コンテンツ増産の仕組み化（最重要・7月の主戦場）

### 1-1. 口コミ投入を「週次ルーチン」に固定する
- **現状**: 8件/3店。単発の職人作業になっている。
- **やること**: 「毎週N件、機会スコア上位ページに本物口コミを投入」を定例化。
  - 対象選定は既存の機会スコア（表示×(5位想定6%−現CTR)）を毎週再計算。
  - `scripts/metrics/` に `weekly_opportunity_scan.mjs` を追加し、GSC APIから自動で「次に書くべきページTOP5」を出す（fetch_metrics.mjsの認証を流用）。
- **効果**: depth>breadth 戦略の実行速度が上がる。7/13の答え合わせで勝ちパターンが確定したら即スケール。

### 1-2. 口コミ投入スクリプトの汎用化
- **現状**: 店ごとに `insert_xxx_review.mjs` を毎回書いている。
- **やること**: `scripts/maintenance/insert_owner_review.mjs` に一本化。
  - 入力: JSONファイル（shop_id / therapist名 / 本文 / ratings / tags / course）
  - 内部: therapist_id のDB自動解決（lessons.md既知の必須事項）・重複チェック・字数判定・--dry-run。
  - therapist未登録なら name-only 登録も同時実行（こころのゆりかご方式）。
- **効果**: 1件あたりの投入コストが激減。ng-rules照合だけに集中できる。

### 1-3. 「答え合わせ」プロトコルの明文化（7/13〜）
- 判定基準を事前に決めておく: 本命3ページの click/順位が動いたか。
  - 動いた → 同店に追加口コミ＋機会スコア次点（spa_flame / aroma_terrace / kawa_spa / number9名古屋24表示）へ横展開。
  - 動かない → 順位反映ラグをさらに2週待ちつつ、タイトル改善（Tier 2-3）を先行。
- metrics-log.md に判定結果と次アクションを必ず記録。

## Tier 2 — SEO構造の底上げ（コンテンツと並走）

### 2-1. エリアページ（/area/:pref）のコンテンツ強化
- **根拠**: hyogo 225表示・saitama 161表示と、サイト最大の表示を持つのに汎用リンク集のまま。
- **やること**（PrefecturePage.jsx / 新設エリアデータ）:
  - エリア概況テキスト（店舗数・主要駅・価格帯の目安）を静的生成。
  - 「このエリアの最新口コミ」セクション（is_public口コミをSSRで3〜5件埋め込み）→ エリアページに一次コンテンツが載る。
  - 市区レベルのサブページ（/area/tokyo/shibuya 等）は**まだ作らない**（薄ページ量産はlessons.mdの空ページ毀損と同じ轍）。
- **効果**: 「〇〇県 メンズエステ」系の汎用語で表示済み→順位を押し上げる素地を作る。

### 2-2. 内部リンクの「口コミハブ」化
- **やること**:
  - 口コミ付きセラピストページ相互リンク: 同店の「口コミがある他のセラピスト」ブロックをThreadDetailPageに追加（SSR）。
  - shopページ→口コミ付きthreadへの導線を最上部に（現在キャストタブの奥）。
  - ホームに「最新の本物口コミ」セクション（is_public限定・SSR）→ クローラーが新規口コミを最短発見。
- **効果**: 口コミページへのクロール頻度・PageRank集中。sitemapだけに頼らない。

### 2-3. タイトル/メタディスクリプションのCTR改善
- **根拠**: 表示の94.7%が4位以下、click偏在。既に表示があるページの`<title>`が汎用形。
- **やること**:
  - 口コミ付きshopページのtitleを「{店名}の口コミ{N}件・セラピスト評判 | メンエスマップ」形式に（件数を入れるとCTRが上がる定石）。
  - descriptionに実口コミの冒頭を動的引用（SSRで生成済みデータを流用）。
- **実装**: `pages/shops/[shopId]/...` のSeoHead呼び出しを口コミ件数連動に。半日仕事。

### 2-4. 構造化データの拡充
- 現状: HealthAndBeautyBusiness + AggregateRating + Review（threads）は済み。
- 追加: **BreadcrumbList**（全階層）と **ItemList**（エリアページの店舗一覧）。リッチリザルトの取りこぼし回収。
- 注意: レビュー構造化データは引き続き「本物口コミのあるページ限定」（既定方針の維持）。

### 2-5. 被リンク・サイテーション獲得（コード外・地味に最重要）
- 現状ほぼゼロのはず。ドメイン権威が無いと順位9位→3位の壁は超えにくい。
- **やること（低コスト順）**:
  - X(Twitter)アカウント開設 → 新着口コミ・エリアまとめを定期ポスト（GA4のreferral獲得＋指名検索育成）。
  - 5ch/therapist系まとめ・個人ブログへの自然言及を狙った「エリア統計コンテンツ」（例: 東京メンエス料金相場マップ）を1本作る＝リンクされる資産。
  - 店舗側への「口コミページできました」連絡（将来のBtoB営業の種まきにもなる）。

## Tier 3 — フライホイール自走化（UGCが自然に増える状態へ）

### 3-1. 投稿導線の最終ピース
- ThreadDetailPage（セラピストページ）に「この人の口コミを書く」固定CTA（現在は店舗経由でStep1から）。therapist_id/shop_idをプリセットしてStep2から開始＝摩擦さらに減。
- 口コミ0件ページには「最初の口コミを書いて全口コミ読み放題◯日」の特別訴求（先行者ボーナスの見せ方だけ変える。付与ロジックは既存）。

### 3-2. 投稿後のリテンション
- 投稿承認時（閲覧権付与時）のメールは実装済み → **「あなたの口コミが◯回読まれました」週次メール**を追加（Resend + 閲覧カウント。テーブル1個追加）。
- 効果: 投稿者の再訪・2件目投稿率を上げる。UGCサイトの定石。

### 3-3. データ鮮度の自動化（中期）
- 退店セラピスト検知: 主要CMSパターン別に「公式サイトの在籍一覧と DB を突合する巡回スクリプト」を月次で回す（既存スクレイパー資産を流用）。
- 在籍が消えたセラピストは非表示でなく「退店済み」表示（口コミ資産はSEO上残す）。
